# Data Platform Capability Assessment – Maturity Model & Scoring Guide

## Purpose
This assessment evaluates the maturity of the organization's data platform across eight capability domains.  
For each domain, individual sub-areas are scored independently and then rolled up into a domain score.  
The domain scores are averaged to produce an overall **Data Platform Maturity Score**.

---

## Maturity Levels

| Level | Label | Description |
|-------|-------|-------------|
| **1** | **Initial** | Capability is absent or entirely ad-hoc. No defined processes, tools, or ownership. Results are unpredictable. |
| **2** | **Developing** | Basic capability exists in isolated pockets. Processes are informal, manually driven, and not consistently repeated. Tooling is minimal or stitched together. |
| **3** | **Defined** | Capability is documented, standardized, and repeatable across teams. A recognized tool or platform supports the process. Ownership and roles are clear. |
| **4** | **Managed** | Capability is measured, monitored, and proactively managed. KPIs/SLAs exist and are tracked. Tooling is integrated and automated. Exceptions are acted upon. |
| **5** | **Optimizing** | Capability is continuously improved using data-driven feedback loops. Leading practices, self-service models, and innovation (e.g., AI/ML augmentation) are embedded. |

**Scoring Tips**
- Score each dimension independently, then derive an overall sub-area score (weighted average or consensus).
- Use the domain-level summary sheet (`summary.md`) to roll up sub-area scores.
- Attach screenshots, architecture diagrams, or links to documentation as evidence where possible.
