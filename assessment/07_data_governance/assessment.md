## 7.1 Data Strategy Management

Data strategy management covers the organization's ability to define, communicate, and execute a coherent data strategy aligned to business objectives.

### Questions

| # | Question | Answer / Evidence |
|---|----------|-------------------|
| 1 | Is there a formally approved data strategy document? Who owns it and when was it last reviewed? | |
| 2 | Is the data strategy aligned to and directly traceable from the overall business strategy and priorities? | |
| 3 | Does the data strategy define target capabilities, investment priorities, and a multi-year roadmap? | |
| 4 | Is there an executive-level data governance body or steering committee that owns the data strategy? | |
| 5 | Are data strategy objectives tracked with measurable KPIs and reviewed on a regular cadence? | |
| 6 | How is the data strategy communicated to and understood by business and technology stakeholders? | |
| 7 | Is there a defined operating model for data (centralized, federated, hybrid / data mesh)? Is it documented? | |
| 8 | Are data investments (tools, people, infrastructure) justified and tracked against strategic outcomes? | |
| 9 | Is the data strategy revisited when major business changes occur (mergers, regulations, new products)? | |
| 10 | Is there a data literacy programme aligned to the strategy to build organisational data capability? | |

## 7.2 Data Standards Management

Data standards management covers the definition, publication, and enforcement of standards governing how data is named, defined, formatted, and used across the organization.

### Questions

| # | Question | Answer / Evidence |
|---|----------|-------------------|
| 1 | Is there a Business Glossary defining key business terms, with agreed definitions, owners, and approval status? | |
| 2 | Are Critical Data Elements (CDEs) identified and documented? How many CDEs are formally managed? | |
| 3 | Are naming conventions defined for databases, tables, columns, and pipelines? Are they enforced (automated checks)? | |
| 4 | Are data classification standards defined (e.g., Public, Internal, Confidential, Restricted) and consistently applied? | |
| 5 | Are data format and encoding standards (date/time, currency, identifiers) documented and enforced? | |
| 6 | Is there a reference data management process governing code sets, lookup values, and master data? | |
| 7 | How are data standards published and made accessible to engineering and business teams? | |
| 8 | Is there a formal change management process for proposing, approving, and communicating updates to standards? | |
| 9 | Are data standards compliance checks built into CI/CD pipelines or data pipeline reviews? | |
| 10 | Who is accountable for maintaining data standards, and how is compliance measured? | |

## 7.3 Data Stewardship

Data stewardship covers the organizational roles, responsibilities, and processes through which individuals are accountable for the quality, security, and appropriate use of data within their domain.

### Questions

| # | Question | Answer / Evidence |
|---|----------|-------------------|
| 1 | Are data steward and data owner roles formally defined with documented responsibilities? | |
| 2 | Are data stewards assigned to all critical data domains? How many active stewards are there? | |
| 3 | Are stewards empowered with tooling (catalog, DQ dashboards, glossary management) to perform their role? | |
| 4 | Is there a stewardship operating model (e.g., regular steward community calls, escalation path, governance forum)? | |
| 5 | How are data quality issues surfaced to and resolved by stewards? Is there a formal issue management workflow? | |
| 6 | Are data stewardship activities tracked and measurable (issues resolved, glossary terms maintained, DQ scores owned)? | |
| 7 | Is stewardship included in job descriptions and performance objectives for relevant roles? | |
| 8 | How are new stewards onboarded and trained? Is there a stewardship playbook? | |
| 9 | Is there a process for managing stewardship gaps when stewards leave or change roles? | |
| 10 | Are stewards engaged in data product definitions, data contract approvals, and change management decisions? | |
